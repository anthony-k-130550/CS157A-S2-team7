How it works

A clean student workflow for your database project.

1
Register or log in Students create an account with name, email, password, and department information.
2
Search for a session Users browse study groups by subject, date, or campus location to find the best match.
3
Join or create one If no session fits, students can create a new one and define title, time, location, and capacity.
4
Manage participation Students can leave a session, while session owners can edit or cancel their own events.

User roles

Two main user groups support the core platform workflow.

S
Student Register, log in, search for sessions, join a session, leave a session, create a session, and manage owned sessions.
A
Admin View students, disable student accounts, review active sessions, remove sessions, and monitor platform statistics.
